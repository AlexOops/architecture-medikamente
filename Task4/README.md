# Task4 — Оценка узких мест при миграции

## Цель

Оценить потенциальные узкие места при миграции устаревшей монолитной и файловой архитектуры «Медикаменте» на современную целевую архитектуру.

## Что подготовлено

- диаграмма Исикавы в формате PlantUML;
- отчёт по выявленным проблемам;
- список рекомендаций с приоритетами.

## Структура

## Диаграмма

Исходник:

```text
Task4/diagrams/ishikawa_migration_bottlenecks.puml
```

## Основной вывод

Главный риск миграции — не сам переход на микросервисы, а перенос неуправляемых данных и процессов в новую архитектуру без предварительной нормализации, классификации, аудита и контроля интеграций.
