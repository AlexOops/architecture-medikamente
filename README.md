# architecture-medikamente

Проектная работа 10 спринта.

Кейс: компания «Медикаменте» — медицинская компания, которая переходит от ручного учёта в Excel, файлового хранения и локальных систем к целевой архитектуре с управляемыми данными, безопасным доступом, интеграциями и аналитическим слоем.

## Цель проекта

Подготовить архитектурные решения для автоматизации процессов компании с учётом:

- защиты персональных и медицинских данных;
- принципов Privacy by Design и Data Minimization;
- контроля потоков данных и Data Lineage;
- масштабирования под новые филиалы, сотрудников и клиентов;
- перехода от Excel/сканов к управляемым системам;
- подготовки платформы для BI, ML и AI.

## Ключевые архитектурные решения

- Ввести классификацию и тегирование данных: `PII`, `PII_EXTENDED`, `MEDICAL`, `PAYMENT`, `LEGAL`, `HR`, `SECRET`.
- Перенести медицинские карты и данные пациентов из Excel/сканов в управляемые системы CRM/EMR.
- Использовать IAM/SSO, MFA, RBAC и ABAC.
- Проверять доступ к чувствительным данным через Policy Decision Point.
- Все внешние интеграции проводить через API Gateway.
- Для лаборатории использовать минимальный набор данных и псевдонимизацию пациента.
- Включить Audit Log для операций с PII и MEDICAL.
- Использовать DLP и SIEM для контроля утечек и инцидентов.
- Для BI/ML/AI использовать отдельный защищённый аналитический слой.
- Не передавать сырые PII и MEDICAL данные в аналитику без тегирования, политики доступа и обезличивания.
- Управлять сроками хранения и удалением данных через Retention & Deletion процесс.
- Миграцию выполнять поэтапно, начиная с аудита данных, IAM, API Gateway и базовых доменов.
