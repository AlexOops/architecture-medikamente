# Task1 — Анализ безопасности системы

## Цель

Проанализировать текущее состояние As-Is архитектуры «Медикаменте» с точки зрения хранения, обработки и передачи конфиденциальных данных.

## Что подготовлено

- Data Flow Diagrams по ключевым процессам.
- Список проблемных зон.
- Матрица защищаемых данных.
- Модель тегирования данных.
- Список мер и инструментов защиты.

## Основные выводы

Текущее состояние несёт высокий риск утечки данных:

- данные пациентов хранятся в Excel, PDF, JPG и сканах;
- нет централизованного аудита доступа;
- нет классификации и тегирования данных;
- нет разделения доступа по ролям и атрибутам;
- медицинские данные находятся рядом с пользователями;
- нет управляемого жизненного цикла данных;
- обмен данными между системами не контролируется.

## Ключевые рекомендации

- Ввести классификацию и тегирование данных.
- Перевести хранение медицинских карт из файлового сервера в защищённое хранилище.
- Использовать RBAC/ABAC для сотрудников.
- Шифровать данные при хранении и передаче.
- Маскировать и обезличивать данные для аналитики, BI, ML и AI.
- Включить аудит доступа к медицинским данным.
- Ввести DLP, SIEM и мониторинг аномалий.
- Для интеграций использовать API Gateway и data contracts.
