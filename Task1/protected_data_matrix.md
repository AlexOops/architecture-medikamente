# Матрица данных для защиты

| Категория данных | Примеры | Где сейчас | Риск | Защита |
|---|---|---|---|---|
| PII базовые | ФИО, дата рождения, телефон, email | Excel, договоры, сканы, журнал пациентов | Идентификация пациента | Шифрование, RBAC, masking в UI |
| PII расширенные | адрес, место работы/учёбы | анкеты, сканы, Excel | Повышенный ущерб при утечке | Шифрование, ABAC, data minimization |
| Медицинские данные | диагнозы, хронические заболевания, назначения | медкарты, PDF/JPG, файлы | Самые чувствительные данные | Шифрование, ABAC, audit log, DLP |
| Медицинские документы | заключения, протоколы приёма, результаты обследований | сканы, PDF, общий диск | Утечка врачебной тайны | Шифрование файлов, контроль скачивания |
| Лабораторные данные | анализы, показатели, дата анализа | Excel-реестры, лаборатория | Раскрытие состояния здоровья | API minimization, pseudonymization |
| Платёжные данные | факт оплаты, сумма, чек, договор | 1С, ККМ, Excel | Финансовые риски | RBAC, audit, шифрование backup |
| Договорные данные | договор, согласия, подписи | сканы, PDF, файловый сервер | Юридические риски | Шифрование, retention policy |
| Учётные данные | логины, группы AD/LDAP | Windows Server / AD | Захват доступа | MFA, RBAC, rotation, audit |
| Данные сотрудников | ФИО, должность, зарплата, кадровые данные | 1С Бухгалтерия | Утечка HR-данных | RBAC, encryption, audit |
| BI/ML/AI datasets | выгрузки из Excel, витрины, датасеты | локальные файлы, Jupyter | Повторная идентификация | Обезличивание, tokenization, lineage |

## Способы защиты

### Шифрование

Использовать для:

- медицинских карт;
- документов;
- сканов;
- резервных копий;
- таблиц с PII;
- соединений между системами.

### Обфускация / маскирование

Использовать для:

- отображения телефона;
- email;
- документов;
- адресов;
- экранов ресепшена;
- логов и BI-отчётов.

### Обезличивание

Использовать для:

- BI;
- ML;
- AI;
- тестовых окружений;
- выгрузок для аналитиков.

### Псевдонимизация

Использовать для:

- лабораторных интеграций;
- аналитических витрин;
- межсистемных обменов;
- случаев, где нужен технический ID без раскрытия ФИО.
