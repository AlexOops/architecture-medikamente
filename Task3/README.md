# Task3 — Стратегия защиты данных

## Цель

Разработать стратегию защиты данных для целевой системы «Медикаменте».

Стратегия учитывает разные состояния данных:

- данные при хранении;
- данные при передаче;
- данные при использовании;
- данные в резервных копиях;
- данные в аналитике, BI, ML и AI.

## Что подготовлено

- классификация данных;
- обоснование необходимости защиты;
- стратегия защиты данных;
- автоматизированный контроль;
- реестр программных и аппаратных средств защиты;
- диаграмма целевого контура защиты данных.

## Ключевой принцип

Данные должны защищаться на всём жизненном цикле:

```text
сбор -> хранение -> передача -> обработка -> аналитика -> архив -> удаление
```

Для каждого класса данных определяется:

- чувствительность;
- состояние данных;
- риск;
- владелец;
- срок хранения;
- допустимый способ передачи;
- способ защиты;
- требования к аудиту.
